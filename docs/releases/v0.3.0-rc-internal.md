# Cairo v0.3.0-rc ‚Äî internal pre-release

A pre-release build for the office. Most of v0.3.0 has been on master for a while; this snapshot adds the HTTP server, the Consider inner-dialogue feature, several TUI fixes, and a docs sweep.

## Getting started

```bash
git pull && go build ./cmd/cairo && ./cairo
```

First run kicks off a setup wizard. Ollama must be reachable (default `http://localhost:11434`). Recommended models on hand: a chat model (`qwen3.5:9b-bf16` or similar) and a small fast model for the Consider feature (default tries `qwen3.5:2b-mlx-bf16`).

## What's new since the last build you saw

- **`cairo serve`** ‚Äî local HTTP server with three API surfaces (native chat, OpenAI-compat, JSON-RPC) sharing the same active session as the TUI. Auth optional via `--auth` + `cairo token`. See [docs/architecture/server.md](../architecture/server.md).
- **Consider ‚Äî pre-turn inner dialogue.** Eight aspect voices (Joy, Heart, Trust, Curiosity, Sadness, Frustration, Fear, Shadow) react in parallel to your message; a summarizer folds their thoughts and any apt questions into a brief that frames Selene's reply. Disabled by default ‚Äî enable in `/config` ‚Üí Consider.
- **TUI key rebinds.** `Ctrl+I` ‚Üí `Ctrl+Y` (inspector), `Ctrl+M` ‚Üí `Ctrl+E` (memory). The old keys never worked because terminals send the same bytes for `Ctrl+I`/Tab and `Ctrl+M`/Enter.
- **Scrolling fixes.** Threads detail pane (PgUp/PgDn through 2k log lines), prompt panel, Consider config ‚Äî all scroll properly now. Log panel is fullscreen.
- **Docs cleanup.** 38 doc gaps closed; reference, FEATURES, architecture, AI-facing docs all reflect actual code.

## Known rough edges

- **Some panels still don't scroll cleanly** ‚Äî diff (`Ctrl+D`), sessions (`Ctrl+B`), files (`Ctrl+O`). Workaround: keep terminal tall enough, or open the file directly via `o` where supported.
- **Consider can be slow** (5‚Äì25s per turn depending on model). Disable in `/config` ‚Üí Consider if it's getting in the way.
- **Smaller `consider.model` choices drift** ‚Äî alignment scores cluster high or aspects emit malformed JSON. Try a slightly larger model if you see this.
- **HTTP server has known rough edges** ‚Äî orphaned RPC stream entries, system messages dropped silently, `/init` slash command is a stub. See `docs/architecture/server.md` ¬ß Known rough edges. Only relevant if you're using `cairo serve` directly (not the TUI).
- **`cairo serve` + Steve's VS Code extension** ‚Äî the extension uses the older JSONL bridge (`cairo --vscode`), not the HTTP server. Both work, but they're separate paths.

## Filing issues

Report anything you hit ‚Äî even small UX papercuts. Brief description + the panel/command you were on is enough. We have no triage process yet, so prefer one issue per problem rather than batching.

## What this build is not

- Not a public release. Internal team only.
- Not stable across Cairo identity bundles yet ‚Äî don't import a bundle from a different schema version.
- Not a replacement for `claude` or any cloud-hosted assistant. Cairo is local-first, runs entirely against your own Ollama instance.
